# CONTRIBUTING.md — Як долучитися / користуватися проєктом

Дякуємо за інтерес до **TodoApp (Tkinter)**. Нижче — правила й поради для роботи з кодом і тестами, а також інструкції для користувачів, які хочуть локально запускати застосунок.

## 1) Як почати (розробникам)
1. **Клонування**
   ```bash
   git clone <repo-url>
   cd <repo>
   python -m venv .venv && . .venv/bin/activate
   pip install -U pip pytest
   ```
2. **Запуск застосунку**
   ```bash
   python todo_buggy.py
   ```
3. **Запуск тестів**
   ```bash
   pytest -q
   ```
   У разі headless-середовища (CI) використовуйте:
   ```bash
   xvfb-run -a pytest -q
   ```

## 2) Стиль коду і комітів
- Python 3.10+, стандартна бібліотека Tkinter
- Дотримуйтеся простого стилю PEP 8 (за бажанням підключіть `ruff`/`flake8`)
- Повідомлення комітів — у стилі Conventional Commits (наприклад, `feat:`, `fix:`, `test:`)
- Не додавайте зайвих залежностей

## 3) Тестування
- Для кожної логічної зміни — щонайменше один тест
- Використовуйте `pytest`:
  - **фікстури** для створення `Tk`/`TodoApp`
  - **parametrize** для різних вхідних значень
  - **raises/skip/xfail** за потреби
- Локальний швидкий прогін: `pytest -q`

## 4) Репорти про помилки та пропозиції
- Створіть **issue** і додайте:
  - кроки відтворення
  - очікувану та фактичну поведінку
  - версію Python/ОС
  - (за можливості) скріншоти

## 5) Pull Request
1. Окрема гілка від `main`/`master`
2. Тема PR: коротко *що* і *навіщо*
3. У PR має проходити `pytest`
4. За потреби оновіть `README.md`

## 6) Для користувачів (довідка з використання)
- Запуск: `python todo_buggy.py`
- Додавання: введіть текст у поле та натисніть «Додати»
- Відмітка виконання: клік по елементу списку
- Редагування: оберіть елемент → «Редагувати»
- Видалення: оберіть елемент → «Видалити»
- Тести: `pytest -q`

## 7) Ліцензія
Освітня/академічна. За потреби оновіть на MIT або іншу.
