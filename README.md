# TodoApp (Tkinter)

Невеликий GUI-застосунок на **Tkinter** для ведення списку справ (to‑do). Підтримує додавання, позначення виконання, редагування та видалення завдань. Проєкт супроводжується автотестами на **pytest**.

> Швидкий старт: `python todo_buggy.py` — запустити застосунок; `pytest -q` — запустити тести.

## Можливості
- Додавання завдань через поле вводу
- Ігнор порожнього/плейсхолдерного вводу
- Позначення виконання (done/undone) кліком
- Редагування тексту обраного завдання
- Видалення завдання
- Збереження та відновлення стану (якщо реалізовано у вашій версії)

## Вимоги
- Python 3.10+
- Стандартна бібліотека `tkinter`
- (Для тестів) `pytest`

## Встановлення
```bash
# (рекомендовано) у віртуальному середовищі
python -m venv .venv && . .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -U pip pytest
```

## Запуск
```bash
python todo_buggy.py
```

## Тести
У репозиторії є окремий файл з новими pytest-тестами:
- `tests/test_todo_app_pytest.py` (у цій сесії — `test_todo_app_pytest.py`)
Запуск:
```bash
pytest -q
```

### Що саме перевіряється
- Додавання валідного завдання
- Ігнор порожнього/плейсхолдерного вводу
- Параметризація додавання різних значень
- Очікуване виключення при хибному виклику (`raises`)
- Пропуск інтерактивного тесту (`skip`)
- Очікувано проблемна поведінка (`xfail`) для відомого дефекту видалення
- Навмисний «падаючий» тест для демонстрації звіту

## Використання
1. Введіть текст завдання у поле вводу та натисніть «Додати».
2. Клікніть по елементу у списку, щоб перемкнути стан «виконано/не виконано».
3. Виділіть елемент і натисніть «Редагувати», щоб змінити текст.
4. Виділіть елемент і натисніть «Видалити», щоб прибрати його.

> Порада (CI/безголовий режим): для запуску тестів Tkinter у CI використовуйте віртуальний дисплей (наприклад, `xvfb-run -a pytest`) або фікстури, що викликають `root.withdraw()`.
